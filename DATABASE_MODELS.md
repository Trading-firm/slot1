# Database Models Documentation

This document describes the structure of the MongoDB collections used by the trading bot. These collections are designed to be consumed by the MERN stack web application for visualization and data manipulation.

## Database Overview
- **Database Name**: `trading_bot` (configured via `MONGO_DB_NAME` in `.env`)
- **Type**: MongoDB

---

## Collections

### 1. `trades`
Stores every trade executed by the bot, including both open and closed positions.

| Field | Type | Description |
| :--- | :--- | :--- |
| `_id` | `ObjectId` | Unique identifier for the trade. |
| `pair` | `string` | The asset pair traded (e.g., "Volatility 10 Index"). |
| `direction` | `string` | Trade direction: `"BUY"` or `"SELL"`. |
| `entry_price` | `double` | Price at which the trade was opened. |
| `stop_loss` | `double` | Stop loss price level. |
| `take_profit` | `double` | Take profit price level. |
| `quantity` | `double` | The size of the position. |
| `broker_order_id` | `mixed` | Unique ID provided by the broker (MT5/Binary.com). |
| `mode` | `string` | Execution mode: `"LIVE"` or `"DEMO"`. |
| `timeframe` | `string` | The timeframe used for analysis (e.g., `"M5"`, `"H1"`). |
| `strategy` | `string` | Name of the strategy that generated the signal. |
| `status` | `string` | Current status: `"OPEN"` or `"CLOSED"`. |
| `exit_pending_count`| `int` | Number of attempts to close the trade (used for error handling). |
| `exit_pending_reason`| `string?` | Reason why an exit is pending or failed. |
| `exit_price` | `double?` | Price at which the trade was closed. |
| `exit_reason` | `string?` | Why the trade closed (e.g., `"TP hit"`, `"SL hit"`, `"Strategy exit"`). |
| `pnl` | `double?` | Realized Profit and Loss in currency units. |
| `pnl_pct` | `double?` | Realized Profit and Loss as a percentage. |
| `entry_time` | `Date` | UTC timestamp when the trade was opened. |
| `exit_time` | `Date?` | UTC timestamp when the trade was closed. |

**Indexes:**
- `status` (ASC)
- `pair` (ASC)
- `entry_time` (DESC)

### 2. `signals`
Stores every signal generated by the bot strategies, whether they were acted upon or skipped.

| Field | Type | Description |
| :--- | :--- | :--- |
| `_id` | `ObjectId` | Unique identifier for the signal. |
| `pair` | `string` | The asset pair analyzed. |
| `signal_type` | `string` | Direction suggested: `"BUY"` or `"SELL"`. |
| `timeframe` | `string` | The timeframe of the analysis. |
| `strategy` | `string` | The strategy that produced the signal. |
| `close_price` | `double` | Price of the asset at signal generation. |
| `ema_fast` | `double?` | Value of the Fast EMA (if applicable). |
| `ema_slow` | `double?` | Value of the Slow EMA (if applicable). |
| `rsi` | `double?` | RSI indicator value (if applicable). |
| `atr` | `double?` | ATR indicator value (if applicable). |
| `bb_upper` | `double?` | Bollinger Band Upper value (if applicable). |
| `bb_lower` | `double?` | Bollinger Band Lower value (if applicable). |
| `acted_on` | `boolean` | `true` if a trade was opened based on this signal. |
| `reason_skipped` | `string?` | Explanation if the signal was not acted upon (e.g., "Max trades reached"). |
| `metadata` | `object` | Additional strategy-specific data. |
| `created_at` | `Date` | UTC timestamp when the signal was generated. |
| `trade_id` | `string?` | The `_id` of the linked trade in the `trades` collection. |
| `pnl` | `double?` | Final PnL of the linked trade (updated when trade closes). |
| `pnl_pct` | `double?` | Final PnL % of the linked trade (updated when trade closes). |
| `outcome` | `string?` | `"WIN"` or `"LOSS"` (updated when trade closes). |
| `exit_reason` | `string?` | Exit reason from the linked trade. |

**Indexes:**
- `pair` (ASC)
- `created_at` (DESC)

---

### 3. `daily_summary`
Aggregated performance snapshots for each day.

| Field | Type | Description |
| :--- | :--- | :--- |
| `_id` | `ObjectId` | Unique identifier. |
| `date` | `string` | The date in ISO format (`"YYYY-MM-DD"`). **Unique Index**. |
| `total_trades` | `int` | Number of trades closed on this day. |
| `winning_trades` | `int` | Number of trades with PnL > 0. |
| `losing_trades` | `int` | Number of trades with PnL <= 0. |
| `win_rate` | `double` | Percentage of winning trades. |
| `total_pnl` | `double` | Sum of PnL for all trades closed on this day. |
| `best_trade_pnl` | `double` | Highest PnL among closed trades. |
| `worst_trade_pnl` | `double` | Lowest PnL among closed trades. |
| `updated_at` | `Date` | UTC timestamp of the last update. |

**Indexes:**
- `date` (ASC, UNIQUE)

---

### 4. `bot_state`
Persistent key-value storage for bot configuration and runtime state.

| Field | Type | Description |
| :--- | :--- | :--- |
| `_id` | `ObjectId` | Unique identifier. |
| `key` | `string` | Unique key for the state item (e.g., `"trading_enabled"`). **Unique Index**. |
| `value` | `string` | The value stored for the key. |
| `updated_at` | `Date` | UTC timestamp of the last update. |

**Indexes:**
- `key` (ASC, UNIQUE)
